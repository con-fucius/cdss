## Absolute rules — read before writing a single line

1. Read before writing. For every file you touch, read its current content first. Never edit based on memory from earlier in a session.
2. No bare 'except:'. Every caught exception has a named variable and a 'logger.warning("...: %s", exc)'. No silent swallowing.
3. Every 'asyncio.create_task()' has '.add_done_callback()' that logs exceptions. Background tasks that fail silently are invisible bugs.
4. No PHI in logs anywhere. 'incident_desc', 'chief_complaint', 'description', patient names — never in any 'logger.*' call. CI must catch this.
5. No external HTTP call propagates exceptions to the incident path. Every external call returns 'None'/empty on failure.
6. Type annotations on every function. 'def f(x)' is wrong. 'def f(x: str) -> Optional[int]' is correct.
7. 'ruff' passes zero warnings before any file is committed.
8. No '.env' files committed. Only '.env.example'.
9. No 'allow_origins=["*"]' in production. 'validate_startup_config()' must enforce this in every service.
10. Clinical content in protocol files and 'clinical_rules.yaml' is reviewed by a clinician before commit. A comment block '# reviewed_by: [name]' is required on guidance notes containing clinical instructions.
11. Protocol files are never committed without reachability validation passing ('_validate_protocol_reachability' in registry).
12. Windows/PowerShell environment. Scripts, paths, and commands must work in PowerShell. No bash-isms.

## Pitfalls and constraints to not violate

| # | Pitfall | Rule |
|---|---|---|
| 1 | spaCy model download at container startup | Model must be in Dockerfile 'builder' stage. 'RUN python -m spacy download en_core_web_md' in build, never in app startup. |
| 2 | Triage enrichment blocking incident creation | Fire as 'create_task()' only. Incident endpoint returns before enrichment resolves. |
| 3 | 'asyncio.create_task()' silent failures | Every 'create_task' has '.add_done_callback(lambda t: logger.error("...", t.exception()) if t.exception() else None)'. |
| 4 | Facility JSON file data store | Postgres only. JSON files cannot support live updates, versioning, or atomic upserts. |
| 5 | BallTree rebuilt on every request | Singleton. Rebuilt only at startup and on explicit '/admin/reload-facilities'. |
| 6 | Geocoding cache miss on every request | TTLCache on 'geocode()'. Repeat location strings (very common in a dispatch centre) cost zero external calls. |
| 7 | 'outcome_route_to_other_protocol' left dangling | Once target protocols exist, remove 'outcome_route_to_other_protocol' from all existing protocol files and replace with specific terminal outcomes. |
| 8 | PHI in logs | No 'incident_desc', 'chief_complaint', 'description', or patient name in any 'logger.*' call in any service. |
| 9 | Swahili terms absent from protocol triggers | Every high-volume protocol must have Swahili 'chief_complaint_trigger' entries. |
| 10 | Clinical rules with no review attribution | Every 'severity_weight > 0.7' entry in 'clinical_rules.yaml' needs '# clinical_basis:' comment. Guidance notes in protocols need '# reviewed_by:' comment. |
| 11 | Stale facility data treated as live | 'data_as_of' always in 'GET /health' and 'GET /data-currency' responses. Dispatcher UI must display it. |
| 12 | Missing migration head check | Before writing any new migration: run 'alembic heads' to confirm single head. New migration's 'down_revision' must match current head exactly. |
| 13 | CORS wildcard in production | 'validate_startup_config()' in every service must block 'ENVIRONMENT=production' if 'ALLOWED_ORIGINS' is unset or '*'. |
| 14 | Admin endpoints unprotected | '_require_admin_key' FastAPI dependency applied to every '/admin/*' route in every service. |
| 15 | ETA as a magic number | Ambulance speed is 'AMBULANCE_SPEED_KMH' from config (default 60). Not hardcoded. Documented: "urban/rural Kenya realistic average." |

- Protocol sign-off: 8 protocols need named medical director + named reviewing clinician in 'approved_by'/'approved_date'
- Facility data source: need a current Kenya/Uganda/DRC facility dataset. KMHFL (Kenya Master Health Facility List) is the right source for Kenya. Data currency cadence must be decided.
- UMLS API key: needed for L3 UMLS resolution in triage ranker. Without it, service runs in degraded mode (L4 fallback only) — still functional but no canonical SNOMED/ICD-10 codes on cache misses.
- 'FACILITY_REGISTRY_BASE_URL' and 'EMERGENCY_DISPATCH_BASE_URL': once the services are running, these must be set in 'ambulance-cdss' '.env' before facility routing and unit dispatch actually work.

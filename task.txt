Goal

'''text

1.1 Multi-disease retrieval resolution
Problem
'_select_retrieval_disease' returns at most one disease ID. Comorbidity queries (e.g. HIV+diabetes) retrieve from one guideline table only, even though 'PatientContext.active_conditions' and 'comorbidity' carry the second disease, and '_infer_disease_from_query' discards ties instead of returning multiple hits.

Tasks
- [ ] New function '_resolve_retrieval_diseases(available, context, query) -> List[str]' returning 1-3 disease IDs.
- [ ] '_infer_disease_from_query': when multiple diseases score equally (or within a small margin), return all of them instead of 'None'.
- [ ] Merge query-inferred diseases with 'context.active_conditions' ∩ 'available' and 'context.comorbidity' (mapped to a disease ID if it matches a configured disease).
- [ ] Cap result at 3 diseases (bound fan-out).
- [ ] 'SearchIndex.search_guidelines': accept 'disease: Optional[List[str]]' (keep 'Optional[str]' working via wrapping in a list internally) — fan out per disease, merge/sort results.
- [ ] '_query_evidence_context': accept the same disease list, query each, concatenate graph context blocks.

Files

- 'HIV-agent/app/search_tools.py'
- 'HIV-agent/app/evidence.py'

Acceptance checklist

- [ ] A query with 'context.active_conditions = ["hiv", "tb"]' retrieves from both tables even if the query text only mentions one.
- [ ] Single-disease queries/contexts behave identically to current output (regression check on existing test cases in 'test_query_routing.py').
- [ ] 3-disease fan-out completes within 'SEARCH_TASK_TIMEOUT_SECONDS'.
